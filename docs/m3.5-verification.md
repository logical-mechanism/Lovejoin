# M3.5 — UI vertical-slice verification

This file records the manual Preprod verification of the M3.5 UI: deposit
+ withdraw round-trip from a CIP-30 browser wallet against the live
Preprod bootstrap (see `artifacts/preprod/addresses.json`).

The slice is intentionally minimal; see `milestones.json` for scope. This
doc is the operator's checklist + record of what was actually executed.

## Prerequisites

- `nvm` with node 22 — VSCode's snap-shim node breaks pnpm 10
  (see [README.md §Develop](../README.md)).
- pnpm 10 (`pnpm install` at the repo root).
- A Preprod CIP-30 wallet (Lace, Eternl, or Nami) with > 15 ada and a
  collateral UTxO available.
- A free Blockfrost project ID for Preprod
  (<https://blockfrost.io>; the same key the CLI consumes).
- `artifacts/preprod/addresses.json` already produced by the M2 bootstrap
  ceremony. The build script copies this file to
  `ui/public/addresses.preprod.json` automatically when present; if the
  bootstrap is re-run, copy the new file in.

## Run

```sh
make ui-dev   # vite on http://localhost:5173
```

In the browser:

1. Paste the Blockfrost project ID into the **Configuration** panel and
   click **Save**.
2. Click your wallet under **Wallet** and approve the connection prompt.
3. **Deposit** — set rounds (default 30) and click Deposit. The wallet
   will prompt for signature; on submission the panel shows the tx hash.
4. The new box appears in **My boxes (this session)** with its label, txId
   prefix, and a copy-secret button. Save the secret somewhere safe — the
   in-session list is wiped on tab close (persistence lands in M6).
5. **Withdraw** — click *Use* next to the box; the form is pre-filled with
   the secret, ref, a, b. Paste a destination address (different from the
   wallet's change address, ideally a Seedelf one) and click Withdraw.

After confirmation on chain, the destination should hold one denomination
of ada (10 ADA per `artifacts/preprod/addresses.json`).

## Known limitations carried into M4

- No Mix builder yet — `buildMixTx` lives in M4. The vertical slice does
  not exercise the externally-supplied collateral path.
- No persistence: closing the tab loses unspent box secrets. Plan an
  IndexedDB layer in M6.
- No i18n lint rule yet (M0 deliverable revisited in M6 polish).
- The Pool screen, Vault screen, and Mix screen are deferred to M5/M6.

## Browser bundling notes

These bundler tweaks are baked into `ui/vite.config.ts`. If a future
mesh upgrade changes the dep graph, verify each is still required:

- **`vite-plugin-wasm` + `vite-plugin-top-level-await`** — mesh's
  `@sidan-lab/sidan-csl-rs-browser` ships a Wasm module that uses the ESM
  integration proposal Vite doesn't natively understand.
- **`vite-plugin-node-polyfills` (events / crypto / stream / util / process)** —
  mesh's provider + cbor stack reaches for these node builtins.
  Buffer is exposed separately via `ui/src/lib/polyfill.ts` so the plugin
  doesn't auto-inject magic shims into sibling workspaces.
- **alias `./libsodium-sumo.mjs` → resolved `libsodium-sumo` package
  entry** — `libsodium-wrappers-sumo`'s ESM build does a relative import
  that breaks under pnpm's strict layout.

If anything breaks at runtime that *typechecks* but explodes in the
browser console, suspect this layer first.
