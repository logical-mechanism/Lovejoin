# 10 — Glossary

| Term                            | Definition                                                                                                                                                                                                                                              |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Mix-box**                     | A UTxO at the script address holding a fixed denomination and an inline `MixDatum {a, b}`. The fundamental object of the protocol.                                                                                                                      |
| **Pool**                        | The set of all unspent mix-boxes at a given denomination's script address.                                                                                                                                                                              |
| **Generation**                  | The number of Mix transformations a particular mix-box has undergone since it was deposited. UI metric only.                                                                                                                                            |
| **Owner secret**                | The discrete-log scalar `x` such that `b = [x]a` for the box's registers. Whoever knows `x` can spend the box via the Owner branch. Distinct from the Cardano wallet key.                                                                               |
| **Re-randomization**            | The Mix operation: `(a, b) → ([y]a, [y]b)` for fresh secret `y`, preserving the relation `b = [x]a`.                                                                                                                                                    |
| **Sigmajoin**                   | The protocol from `papers/sigmajoin.pdf`. An outsourceable variant of Zerojoin.                                                                                                                                                                         |
| **Zerojoin**                    | The protocol from `papers/zerojoin.pdf`. The predecessor; uses two box types (half-mix, full-mix) and requires both parties online.                                                                                                                     |
| **proveDlog(g, u)**             | Zero-knowledge proof of knowledge of `x` such that `u = [x]g`. Schnorr proof.                                                                                                                                                                           |
| **proveDHTuple(g, h, u, v)**    | Zero-knowledge proof of knowledge of `x` such that `u = [x]g` AND `v = [x]h`.                                                                                                                                                                           |
| **Sigma-OR**                    | Composition of N sigma protocols proving "I know a witness for one of these N statements" without revealing which. Paper Appendix C, generalized to arbitrary N for the variable-width Mix construction.                                                |
| **N (mix width)**               | Number of inputs/outputs in a Mix tx. `2 ≤ N ≤ max_n`, where `max_n` is calibrated empirically (initial: 6). Larger N = more privacy per round = fewer txs to reach target privacy.                                                                     |
| **`max_n`**                     | Upper bound on Mix tx width, set in `ProtocolParams`. Determined by Cardano's per-tx script-cost limits; calibrated at M2 stress test.                                                                                                                  |
| **Collateral provider**         | A third-party service (e.g. [giveme.my](https://giveme.my/)) that supplies the Cardano-required collateral input for a tx without requiring the submitter to put up their own. Used by lovejoin's Mix txs to keep the submitter's wallet out of the tx. |
| **Collateral input**            | Cardano protocol requirement: every Plutus tx must have a key-witnessed UTxO that gets seized if any script fails. Forfeited only on script failure; otherwise returned via `collateral_return`.                                                        |
| **Fiat-Shamir transform**       | Method to make an interactive sigma protocol non-interactive by hashing the prover's commitment to derive the verifier's challenge.                                                                                                                     |
| **`ctx`**                       | The context-binding string mixed into the Fiat-Shamir hash to bind a proof to its specific spending transaction. Prevents replay and front-running.                                                                                                     |
| **DDH**                         | Decisional Diffie-Hellman problem. Distinguishing `(g, [a]g, [b]g, [ab]g)` from `(g, [a]g, [b]g, [c]g)` for random `a, b, c`. Hard in BLS12-381 G1's prime-order subgroup.                                                                              |
| **BLS12-381 G1**                | A prime-order subgroup of the BLS12-381 elliptic curve. Plutus V3 has builtin operations for it. We use it for all sigma protocols.                                                                                                                     |
| **CIP-30**                      | Cardano wallet web standard. The browser API used by mesh to talk to user wallets.                                                                                                                                                                      |
| **mesh**                        | The Cardano TS SDK we use for transaction building and CIP-30 wallet integration.                                                                                                                                                                       |
| **ogmios**                      | A WebSocket bridge to cardano-node's mini-protocols. We use it for chainsync and tx submission.                                                                                                                                                         |
| **db-sync**                     | Cardano's PostgreSQL indexer of chain history. We use it for historical queries.                                                                                                                                                                        |
| **Aiken**                       | The contract language we write the validator in. Compiles to Plutus V3.                                                                                                                                                                                 |
| **Half-mix box / Full-mix box** | Zerojoin terminology, for two distinct box types in a mix round. Sigmajoin replaces both with the single mix-box. NOT used in this implementation.                                                                                                      |
| **Fee contract**                | The shared on-chain UTxO pool that pays for Mix tx fees. Sharded across ~10 long-lived UTxOs. Each deposit replenishes one shard with `N · MAX_FEE_PER_MIX` lovelace. Validator: `fee_contract`, with two redeemer paths (`PayMixFee`, `Replenish`).    |
| **Fee shard**                   | One of the ~10 fee-contract UTxOs. Shards exist for concurrency (multiple in-flight txs) and robustness (one drained shard doesn't block others).                                                                                                       |
| **PayMixFee**                   | The fee contract redeemer used in a Mix tx. Validator checks: ≥ 2 mix inputs, exactly 1 fee input/output, no datum change, `tx.fee = (in − out) ≤ MAX_FEE_PER_MIX`, no native assets.                                                                   |
| **Replenish**                   | The fee contract redeemer used in a Deposit tx. Validator checks: 1 fee input/output, no datum change, output value strictly greater than input, no native assets.                                                                                      |
| **MAX_FEE_PER_MIX**             | Config-defined cap on how much ADA a single Mix tx can pull from the fee contract. Calibrated empirically at M2 stress test.                                                                                                                            |
| **Hyperstructure**              | A protocol whose on-chain state is permissionless, immutable, and self-sustaining. No governance, no admin, no off-switch. The lovejoin contracts are designed as a hyperstructure anchored by the reference NFT.                                       |
| **Reference holder**            | Always-False validator that holds a single permanent UTxO containing the protocol NFT and the `ProtocolParams` inline datum. Other validators read params from this UTxO via `reference_inputs`.                                                        |
| **Reference NFT**               | One-of-one token minted at bootstrap by a one-shot policy. Identifies the canonical reference UTxO. Validators use the NFT's `(policy_id, asset_name)` as their compile-time parameter.                                                                 |
| **One-shot mint**               | A minting policy parameterized by a specific UTxO ref; can fire only in the tx that consumes that UTxO. Used to mint the reference NFT exactly once.                                                                                                    |
| **RFC 6979**                    | A standard for deriving cryptographic nonces deterministically via HMAC, originally for ECDSA. We use it for Schnorr/DH-tuple Fiat-Shamir nonces. Removes RNG dependence at proof time.                                                                 |
| **Mixer-bot**                   | A future off-chain process that automates Mix txs continuously. Not in v1; v1 has every user mixing manually via the UI.                                                                                                                                |
| **Outsourceability**            | Sigmajoin's defining feature over Zerojoin: a third party can mix on a user's behalf without the user being online. In v1 the "third party" is just any other user clicking the mix button.                                                             |
| **Seedelf**                     | A separate Cardano stealth-wallet contract ([github.com/logical-mechanism/Seedelf-Wallet](https://github.com/logical-mechanism/Seedelf-Wallet)). Recommended destination for lovejoin withdrawals when the user wants full wallet-side anonymity.       |
| **Stealth withdraw**            | Paper §4.2 concept. Out of scope for lovejoin v1; achieved at the wallet layer via Seedelf instead.                                                                                                                                                     |
| **Stealth payment**             | (Future) Paper §5.2. Pay a mixer service fee in a way that doesn't reveal the mixer's address.                                                                                                                                                          |
| **Anonymity set**               | The set of users a passive observer cannot distinguish you from. After `n` Sigmajoin mixes, `≈ 2^n`.                                                                                                                                                    |
| **Reference script**            | CIP-33. A script published once on-chain and referenced by tx-id in subsequent txs, avoiding embedding the full script bytes. We publish both validators as reference scripts at bootstrap.                                                             |
| **Preprod**                     | Cardano's testnet that mirrors mainnet parameters. Primary target network for v1.                                                                                                                                                                       |
