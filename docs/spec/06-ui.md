# 06 — UI

## Stack

- **React 19** + **Vite** + **TypeScript**.
- **Tailwind v4**.
- **`@meshsdk/react`** for CIP-30.
- **`@tanstack/react-query`** for backend data.
- **`@lovejoin/sdk`** for crypto, tx building, ownership scanning.
- **`react-i18next`** for i18n from M0.

No SSR. Pure SPA.

## Key storage

Encrypted IndexedDB with Argon2id-derived key from a user passphrase. Same pattern as Lace, Eternl, Nami. Optional plaintext export/import.

The SDK exposes storage as an interface; the UI's IndexedDB-backed implementation is the default; tests substitute an in-memory implementation.

## i18n from day one

Strings keyed; English canonical in `en.json`. Lint rule rejects raw English in JSX components. Initial language list grows as community translations land. Number/date formatting respects locale.

## Screens

### Home
- 60-second explainer.
- Network status: pool size, fee-contract balance, collateral provider status, indexer lag.
- "Connect wallet" button.

### Deposit
1. Slider for `N` (mix-rounds; presets 20/30/50; default 30; minimum 5).
2. Tx preview with per-output explanation.
3. Generate box-ownership secret in browser; save-NOW UX (copyable hex, downloadable JSON, optional encrypted IndexedDB).
4. ConfirmModal gating on "I saved this secret."
5. Wallet signs and submits.
6. Watch for confirmation.

### Pool (the user-as-mixer screen)

- Headline: pool size + fee-contract balance + estimated mixes available + collateral provider status.
- Prominent **"Mix N random boxes"** button. Clicking:
  - Picks `N` boxes uniformly at random (`N` = max width the pool supports, configurable via slider).
  - Picks a fee shard at random.
  - Requests collateral from the provider.
  - Builds and submits the Mix tx.
  - Returns confirmation.
- The MixWidthSlider lets the user choose `N` (range 2..max_n); default max. Higher N = better privacy gain per round but larger tx.
- Below: recent mix activity (anonymized).
- Advanced disclosure: pick boxes manually; automate (run continuously while tab is open).
- Mix button is **always available** to all users — even those who haven't deposited. New users can be mixers from day one and that strengthens privacy for everyone.
- Cooldown to prevent spam (5s client-side; backend rate-limits).

### Vault
- Boxes the user owns, derived by scanning the pool with each saved secret.
- Per-row: id, denom, generation, last mixed, withdraw button.
- Import box secret button.

### Box
- Detail page for a single owned box.
- Generation count + parent transactions.
- Withdraw form.
- Estimated linkage probability; explainer text.
- `SeedelfHint` if the destination is a regular Cardano address.

### Withdraw
1. User enters destination address (bech32 validated).
2. UI detects Seedelf addresses heuristically:
   - Seedelf detected: green "Stealth address — wallet identity hidden after withdraw."
   - Regular: yellow `SeedelfHint`.
3. UI shows tx preview (only the wallet input requires a signature; the mix-box is spent by Schnorr proof).
4. Wallet signs.
5. SDK builds Schnorr proof, inserts into redeemer.
6. Submit.

## Privacy UX rules

Non-negotiable:

1. Never display a user's box secret next to identifying info.
2. Local storage opt-in, default off.
3. Pool scans run client-side only.
4. No analytics, no telemetry.
5. No cookies, no tracking IDs. Backend logs IPs only for rate limiting; retention < 24h.
6. Fee-payment wallet is visibly distinct from the box.
7. Mix tx submission shows the fee + collateral source explicitly: "This tx is paid by the fee contract; collateral provided by giveme.my; your wallet is not involved."
8. If the collateral provider is unreachable, surface a yellow banner and **disable the Mix button** until the provider recovers. No default fallback to wallet-collateral — that would defeat the whole point of paying for collateral externally. Users running their own provider can configure it via the SDK if they want fault tolerance against any single service.

## Error handling

- **Recoverable user errors** (invalid address, insufficient funds): inline form errors.
- **Network errors** (backend down, ogmios lag, collateral provider down): top banner; doesn't block local-only flows.
- **Crypto / contract errors**: full-screen with diagnostic + bug-report link. Should be impossible in production.

## Accessibility

- Keyboard-accessible interactive elements.
- `prefers-reduced-motion` respected.
- 9th-grade reading level except advanced screens.

## Build & deploy

- Vite production build → static files.
- Local dev: `pnpm dev` → `http://localhost:5173`, talks to backend at `http://localhost:3001`.
- Single deploy target in v1: a static directory served by the backend or any nginx.
- Anyone can self-host the UI. Hyperstructure principle.
- Hosting plan deferred (OQ-L; likely DigitalOcean alongside the backend).
