## KAT vectors (variable N)

Generated by a Rust reference implementation using `blst`. JSON in `crypto/test-vectors/`:

```
{
  "name": "sigma_or_n4_001",
  "N": 4,
  "a": "0x...", "b": "0x...",
  "outputs": [
    { "a": "0x...", "b": "0x..." },
    { "a": "0x...", "b": "0x..." },
    { "a": "0x...", "b": "0x..." },
    { "a": "0x...", "b": "0x..." }
  ],
  "real_branch_index": 2,
  "witness": "0x...",
  "ctx": "0x...",
  "expected_proof": { "branches": [...] },
  "valid": true
}
```

200 vectors per N for N ∈ {2, 3, 4, 6, 8}. Plus negative vectors at every N.

Each vector verified by:
1. TS prover (re-derive proof using RFC 6979; assert exact bytes).
2. TS verifier.
3. Aiken validator in simulator.

Negative vectors:
- t = identity element
- z = 0 / z = r
- t mutated by 1 byte
- Proof correct but wrong ctx
- Proof correct but wrong public element
- Proof for branch i resubmitted as proof for branch j ≠ i
- N-way proof submitted to validator expecting M ≠ N branches

All three verifiers MUST reject every negative vector.

## Stress test (resolves OQ-A and `max_n` calibration)

`stress-tests/`:

- `fee-calibration.ts`: submit Mix txs with worst-case datum & redeemer sizes at the chosen `max_n`. Capture script CPU/mem; compute Cardano-charged fee; recommend `max_fee_per_mix = ceil(max_observed × 1.25)`.
- `max-n-calibration.ts`: sweep `n ∈ {2, 3, 4, 6, 8, 10}`. Record per-script and per-tx CPU/mem at each. Recommend `max_n` as the largest value where total tx CPU stays < 70% of mainnet limit.

Results committed to `docs/perf.md` and `network.preprod.json`. These run once at M2 close and again after any contract optimization.

## Property tests

- For random `(a, b)` with `b = [x]a`: Schnorr proof always verifies.
- For any byte tampering: verification fails.
- For random `(g, h, x)`: DH-tuple proof verifies.
- For sigma-OR proof at any N from any branch: verifies AND no statistical bias by branch (10k samples per N).
- For random valid Mix tx at any N ∈ {2..max_n}: validator accepts.
- For random tampering of: input datum / redeemer / output datum / output value / script hash / reference UTxO / NFT identifier / proof branch count: validator rejects.

## Fuzz / negative

`tests/fuzz/`:
- Random bytes as proofs at random N.
- Random valid-shape proofs with random inputs.
- Structurally valid txs with semantic violations.

Validator MUST never crash. Spurious accept = P0.

Nightly 30-min run.
